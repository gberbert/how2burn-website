**[2026-06-02T05:33:00-03:00]**
- **O que foi feito:** Correção do esticamento das imagens/mockups do iPhone e Apple Watch no layout mobile em `style.css` substituindo `height` fixo por `aspect-ratio` e `max-width`.
- **Motivo (O Porquê):** O layout Flexbox em telas menores estava comprimindo horizontalmente os mockups (`width`) para caber na tela, mas como a altura (`height`) era fixa, as skins e as imagens ficavam esticadas/distorcidas verticalmente. O uso de `aspect-ratio` garante que a proporção original do mockup seja perfeitamente mantida não importa o tamanho da tela.
- **Snapshot de Segurança:** As propriedades antigas continham `width` e `height` explícitos. Exemplo: `.showcase-mockup { width: 300px; height: 610px; }` e `@media (max-width: 500px) { .showcase-mockup { width: 250px; height: 510px; } }`.

**[2026-06-02T05:43:00-03:00]**
- **O que foi feito:** Refatoração fina dos mockups utilizando `container-type: inline-size` e medidas relativas `cqi` em vez de `px` para as bordas (`border-width`), `border-radius`, e dimensões internas (ex: Dynamic Island, Botões Laterais).
- **Motivo (O Porquê):** O uso de `aspect-ratio` corrigiu o esticamento da imagem, mas os valores em `px` fixos da borda (`14px solid`) e raio da borda (`50px`) não escalavam. Quando o dispositivo encolhia no mobile, esses elementos fixos tomavam uma proporção gigante da tela, fazendo os contornos parecerem "muito grossos" e distorcidos. Transformar em proporção de container (`cqi`) resolve permanentemente a matemática em qualquer tamanho de tela, agindo de forma idêntica a um `transform: scale()`.
- **Snapshot de Segurança:** As propriedades em `.iphone-mockup`, `.iphone-mockup .dynamic-island` e `.watch-mockup` continham `px` estáticos (ex: `border: 14px`, `border-radius: 50px`, `width: 110px`).

**[2026-06-02T07:33:00-03:00]**
- **O que foi feito:** Criação do arquivo de versionamento local (`versionamento.md`) e injeção do número de versão `<span class="site-version" style="opacity: 0.5; margin-left: 10px;">v.1.0.1</span>` nos rodapés (`.footer-bottom`) de todas as páginas HTML do site (`index.html`, `privacy.html`, `terms.html`, `support.html`).
- **Motivo (O Porquê):** O usuário solicitou um sistema de versionamento visual (no footer) para validar de forma rápida se a versão em cache no navegador/dispositivo corresponde à versão deployada, mitigando ambiguidades em testes de QA.
- **Snapshot de Segurança:** Antes da injeção, os rodapés terminavam em `<p>&copy; 2026 How2Burn. Designed with a focus on health.</p>`.

**[2026-06-02T07:38:00-03:00]**
- **O que foi feito:** Substituição das unidades `cqi` por `calc()` com variáveis nativas `--scale` no `style.css` para a construção geométrica de todos os mockups. A versão no rodapé foi atualizada para `v.1.0.2`.
- **Motivo (O Porquê):** O uso do `container-type: inline-size` nos mockups fez com que as unidades `cqi` não medissem a largura real do celular em pixels, mas sim a largura da tela inteira (viewport). O resultado foram bordas e radius extremamente inflados em dispositivos móveis, além de transformar o Watch em um círculo distorcido e de causar barras pretas na imagem (que foi encolhida enquanto os contornos "inflavam" sobre ela). O uso de variáveis CSS `--scale` é blindado e matematicamente estanque contra redimensionamentos de containers pais.
- **Snapshot de Segurança:** Propriedades removidas: `border-radius: 16.66cqi;`, `container-type: inline-size;`. Foram substituídas por `--scale: 0.833; border-radius: calc(50px * var(--scale));` nas lógicas das media-queries.

**[2026-06-02T07:47:00-03:00]**
- **O que foi feito:** Correção do bug de letterboxing (barras pretas) nas imagens dos mockups no Safari Mobile e adição do `object-position: top center`. A versão no rodapé foi atualizada para `v.1.0.3`.
- **Motivo (O Porquê):** No iOS Safari, o motor do WebKit falha ao aplicar `object-fit: cover` em imagens absolutas se o container pai (`.screen`) não tiver uma altura explícita (mesmo que o avô tenha `aspect-ratio`). Isso fazia a imagem colapsar para a sua proporção intrínseca, criando barras pretas. A solução foi mudar `.device-mockup .screen` para `position: absolute; top: 0; bottom: 0; left: 0; right: 0;` forçando o cálculo exato da área de preenchimento. O `object-position: top center` foi adicionado para garantir que a barra de status do iOS das capturas nunca seja cortada. *Hotfix:* Removida a sobreposição maldita de `.screen.carousel { position: relative; }` que estava anulando a posição absoluta e colapsando a altura da tela para 0px (tela preta).
- **Snapshot de Segurança:** O CSS `.device-mockup .screen` foi alterado de `width: 100%; height: 100%;` para `position: absolute; top: 0; left: 0; width: 100%; height: 100%; z-index: 1;`. O CSS `.device-mockup img` ganhou a propriedade `object-position: top center;`. O bloco `.screen.carousel { position: relative; }` foi apagado.

**[2026-06-02T08:08:00-03:00]**
